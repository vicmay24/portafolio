# Experiencia en Proyectos Front-End
He participado en el desarrollo de múltiples aplicaciones web, orientados a distintos sectores como:

- Plataforma para escuela privada
- Sistemas gubernamentales
- Plataforma e-commerce

---

## Enfoque de Arquitectura

Desde los primeros proyectos trabajé bajo principios de Clean Architecture (arquitectura hexagonal), organizando el código con separación clara de responsabilidades para:

- Mejorar la mantenibilidad
- Facilitar la escalabilidad
- Evitar acoplamientos innecesarios entre lógica y presentación

Esto permitió que el crecimiento del proyecto fuera más ordenado y sostenible en el tiempo.

---

## Trabajo en Equipo y Colaboración

En proyectos colaborativos:

- Coordinación constante con el equipo backend para integración de APIs.
- Uso de Postman, Swagger y Hoppscotch para validar respuestas y probar endpoints antes de integrarlos en el front-end.
- Gestión de ramas mediante Git.
- Uso de Pull Requests para revisión y control de cambios.
- Participación en procesos de merge cuidando la estabilidad del proyecto.

---

## Desarrollo Técnico Implementado

Dentro de los proyectos desarrollé:

- CRUDs completos.
- Tablas dinámicas con buscadores y filtros.
- Integración y consumo de APIs REST.
- Manejo de imágenes y carga de archivos.
- Componentes reutilizables y modulares.
- Implementación de pop-ups, pestañas y layouts estructurados.
- Diseño completamente responsivo (desde dispositivos móviles hasta escritorio).
- Uso de ESLint para mantener consistencia y calidad en el código.
- Documentación y comentarios para facilitar el mantenimiento futuro.

---

## Forma de Trabajo

Me enfoco en:

- Escribir código claro y estructurado.
- Priorizar reutilización sobre duplicación.
- Mantener consistencia visual y técnica.
- Cuidar la experiencia del usuario en distintos dispositivos.
- Trabajar de forma colaborativa manteniendo comunicación técnica clara.
